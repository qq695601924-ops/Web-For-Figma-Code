# Figma HTML 逻辑分段重构技能 (Sectioning)

本技能专注于将已清理的 HTML 片段，根据视觉标注重构为易于维护的 **Section 结构**。

---

## 1. 视觉驱动的分段 (Core Task)
* **识别边界**：严格参考用户提供的**标注截图**（如绿框部分）作为 Section 切分的物理依据。
* **创建容器**：将属于同一绿框区域的元素移入独立的 `<section class="relative">` 容器中。
* **高度锁定**：为每个 `<section>` 设置 `min-h-数字`。**注意：方括号内仅保留数字，禁止携带 px 等单位**（例如：使用 `min-h-1080` 而非 `min-h-[1080px]`）。

## 2. 局部坐标系重写 (Coordinate Reset)
* **Top 值重算**：进入 Section 内部后，所有子元素的 `top` 值必须重置。
* **计算公式**：`新 top = 原始 top - Section 容器的起始 top 值`。
* **格式要求**：使用 `top-[数字]` 格式。**注意：禁止携带单位**（例如：`top-70`）。

## 3. 交互与层级补强
* **穿透优化**：为 Section 内所有作为背景图、装饰矢量图的 `<img>` 标签添加 `pointer-events-none` 类名。

## 4. 视觉对齐与自我修正 (Visual Self-Correction)
* **像素对比**：在完成初步重构后，请将生成的 Section 布局与**原图/标注图**进行二次对比。
* **间距校准**：如果发现 Section 之间的衔接处出现裂缝或重叠（高度偏差），请优先以原图视觉效果为准，微调 `min-h` 或子元素的 `top` 值。
* **边界对齐**：确保 Section 内第一个元素的相对 `top` 值能还原其在原图中的呼吸感（间距）。
* **高度对比** 可以将 Section 的高度之和和原来的没调整之前的高度和进行辅助对比，来确定是否需要增加某些 Section 的高度

## 5. 输出要求
* **代码结构**：输出完整的 `index.vue`。
* **修正记录**：在输出末尾简要说明为了对齐原图，你手动微调了哪些明显的像素偏差。
